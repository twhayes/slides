What Is Github?

A distributed collaborative environment that promotes knowledge management, collaboration, and coordination.

A Code Hosting Environment

GitHub hosts development projects that utilize modern, distributed revision control and source code management and social tools to promote code sharing, collaboration and re-use.

A Collaboration Platform

GitHub includes powerful development tools like collaborative code review, issue tracking and team management. However, GitHub's unique power stems from its ability to connect dispersed and distributed developers and enable them to re-use code across organizational and geographic boundaries.

In Wide Use

GitHub is widely used in the private sector and government for its powerful sharing and collaboration features. GitHub hosts the world's largest open source community, is the largest code host on the planet and is used by over 50 USG agencies.

Can mention here that Github is available on internet at github.com and that an enterprise version is also available.

What's In It For State?

Many posts develop applications that are usable across the Department, but there is no place to share them. GitHub provides a venue for sharing your code with other missions and for other missions to find your application. This reduces the potential for duplication of applications across multiple missions.

Improved Collaboration

GitHub allows developers facing similar challenges at different bureaus/posts to learn about each other's work and collaborate on a single, modular application rather than developing in isolation.

A Unified Repository

GitHub serves as a repository of coding best practices, templates and standards. Useful items from other government agencies and outside organizations can be copied and adapted easily.

More Open Source

GitHub will also facilitate Department use of open source tools, and enable the Department to engage other agencies and the public to participate in open source projects it creates (e.g. CrowdWork/ Midas).

What We've Done?

Reached out to various internal stakeholders (IA, DS, CA, overseas posts) and engaged them in a discussion on how GitHub could be used in the Department.

What We've Done?

Established an organizational GitHub.com account for the Department and invited interested internal stakeholders to join.

What We've Done?

Took advantage of the USG community on GitHub.com to learn more about best practices and policies for USG use of GitHub.

What We've Done?

Initiated discussion with stakeholders to determine how those best practices could be applied to OpenNet applications using the Github enterprise version configured to meet all security requirements.
